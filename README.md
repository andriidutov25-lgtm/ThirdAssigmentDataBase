Робота над третім практичним завданням розпочалася з підготовчого етапу, на якому я
виправив помилки підключення в Python-скрипті. Змінив назву схеми на нижній
регістр та додав встановлення search_path(оскільки в мене була схема не public),
завдяки чому скрипт успішно підключився до бази даних і заповнив таблиці
початковими даними без помилок.

У першому завданні я створив SQL-функцію calculate_order_total, яка приймає
ідентифікатор замовлення та розраховує його загальну вартість. Для цього функція
використовує агрегацію SUM для множення кількості товарів на їхню ціну з таблиці
order_items, а також обгортає результат у функцію COALESCE, щоб коректно
повертати нуль для порожніх замовлень.

Друге завдання полягало у написанні процедури create_order для створення нових
замовлень.

Третє завдання вимагало створення процедури add_product_to_order для додавання
товарів у кошик клієнта. Ця процедура виконує одразу кілька важливих кроків:
забороняє додавати нульову або від'ємну кількість, перевіряє наявність достатнього
залишку на складі, фіксує актуальну ціну товару в замовленні та віднімає куплену
кількість від загального залишку в таблиці продуктів.

У четвертому завданні я розробив тригер автоматичного перерахунку сум та
відповідну тригерну функцію. Цей механізм спрацьовує щоразу після додавання,
оновлення або видалення товарів у замовленні, визначає правильний ідентифікатор
чека за допомогою системних записів NEW або OLD, після чого викликає функцію
calculate_order_total  і миттєво оновлює загальну суму в головній таблиці замовлень.

П'яте завдання було створення тригера аудиту для таблиці замовлень. Цей тригер
налаштований виключно на подію створення нового запису, тому щойно процедура
генерує нове замовлення, він автоматично перехоплює його ідентифікатор та
ідентифікатор клієнта, записуючи ці дані у таблицю історії order_log зі статусом
створення нового замовлення.

Фінальне шосте завдання показало всі ці етапи у єдиному послідовному SQL-скрипті
для тестування.
